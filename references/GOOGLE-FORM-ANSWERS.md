## Q: Long-form description (use case, target user, why it matters)
```
ReppS is the missing meta-layer in the agent economy. Every other ASP *does* work — ReppS *checks* work.

Use case: Before an AI agent executes a non-trivial action (swapping tokens, sending a message, hiring another agent, deploying a contract, transferring funds), it calls ReppS. ReppS returns:
- A verdict: PROCEED, REVIEW, or ABORT
- A confidence score (0.0–1.0)
- A list of specific risks
- Alternative interpretations
- Adversarial questions the agent should have asked itself
- A citable check_id for audit logs

Target users: AI agent developers, DeFi trading bots, social media agents, agent orchestrators, compliance teams. The 57% of enterprises that have watched agents be confidently wrong. The 78% of agent pilots that fail in production.

Why it matters: 95% of AI spending has not produced measurable business returns (MIT, 2025). The bottleneck is not model capability — it's the absence of a structural pre-flight check. ReppS is that check.

Pricing: free tier (3/day per IP), $0.01 per full challenge, $0.05 for a 5-challenge bundle, $0.02 for an audit log. All via x402 v2 on X Layer (network eip155:196), asset USDT0. No accounts, no API keys, no subscriptions.

Built as a pure Node 18 + Express ASP. No external dependencies. Rule engine handles structured risks (burn addresses, slippage, honeypot patterns, scam messages, unaudited contracts). Optional LLM upgrade adds deep adversarial reasoning when OPENAI_API_KEY is set. Works either way.
```
